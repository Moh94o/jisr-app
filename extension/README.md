# Jisr Muqeem Sync — إضافة Chrome للدخول التلقائي

تنقل جلسة منصة مقيم تلقائياً إلى Jisr وتعيد الدخول كل ~50 دقيقة بدون أي تدخّل يدوي.

## المميزات

- ✅ **التقاط الجلسة:** يرصد كل طلب API يطلق من مقيم → يستخرج الـ JWT والكوكيز → ينشرها لـ Supabase
- ✅ **الدخول التلقائي:** كل ~50 دقيقة (قبل انتهاء الجلسة بـ 10 دقائق) يفتح مقيم في تبويب خفي ويسجّل دخول تلقائياً
- ✅ **OTP تلقائي:** يسحب آخر OTP من جدول `otp_messages` في Jisr (الموجود حالياً) ويملأه تلقائياً
- ✅ **بدون lookup يدوي:** Jisr يقرأ الجلسة من Supabase كل 30 ثانية تلقائياً

## التركيب (مرة واحدة)

1. افتح `chrome://extensions` في Chrome
2. شغّل **Developer mode** أعلى يمين الصفحة
3. اضغط **Load unpacked** → اختر مجلد `extension/`
4. الإضافة بتظهر باسم "Jisr Muqeem Sync"

## الإعداد (مرة واحدة)

1. اضغط أيقونة الإضافة في شريط الأدوات
2. افتح **"إعدادات الدخول التلقائي"**
3. أدخل:
   - **اسم المستخدم:** `7049541498` (أو أيّاً كان حسابك في مقيم)
   - **كلمة المرور:** `Refah@2030`
   - فعّل **تفعيل الدخول التلقائي**
4. اضغط **حفظ الإعدادات**
5. اضغط **دخول تلقائي الآن** للاختبار الفوري

## كيف يشتغل

```
كل دقيقة:
  ├─ يفحص هل الجلسة منتهية أو ستنتهي خلال 10 دقائق؟
  └─ إذا نعم: يفتح muqeem.sa/#/login في تبويب خفي
     ├─ Content script يكتشف نموذج الدخول
     ├─ يملأ اسم المستخدم + كلمة المرور
     ├─ يضغط "دخول"
     ├─ يكتشف صفحة OTP
     ├─ يستعلم Supabase RPC: get_latest_muqeem_otp
     ├─ يملأ OTP + يضغط "تحقق"
     ├─ ينتقل لصفحة "البحث عن مقيم لا يعمل للمنشأة"
     ├─ Background.js يلتقط Authorization header من أول طلب API
     ├─ ينشر الجلسة لـ Supabase
     └─ يقفل التبويب
```

## الحالات

- **شارة ✓ خضراء** على الأيقونة: الجلسة نشطة
- **شارة • زرقاء**: جاري الدخول التلقائي
- **شارة ! حمراء**: خطأ — افتح الـ popup للتفاصيل
- **بدون شارة**: لم يتم الدخول بعد

## استكشاف الأخطاء

افتح `chrome://extensions` → الإضافة → **service worker (inspect)** للـ background logs.

أو: افتح Muqeem في تبويب → F12 → Console — سترى logs بـ `[Jisr]`.

## ملاحظات أمان

- بيانات الدخول تُخزَّن في `chrome.storage.local` فقط على جهازك (مشفّرة بـ Chrome)
- الجلسة (JWT) محفوظة في Supabase تحت RLS مقيّد
- الـ OTP يُسحب عبر RPC مخصصة (`get_latest_muqeem_otp`) — لا تكشف باقي الرسائل
- لا يوجد إرسال بيانات لأي طرف ثالث

## محدوديات

- لو reCAPTCHA طلب checkbox (نادر للمتصفحات الموثوقة) → فشل الدخول التلقائي. تحتاج تسجّل دخول يدوي مرة واحدة.
- لازم Chrome مفتوح طوال الوقت لتشتغل الـ alarms.
- مقيم قد يطرد جلسات متزامنة لو سجّلت دخول من جهاز آخر بنفس الحساب.
