# Jisr WhatsApp Invoice Bot — بوت فواتير الواتساب

بوت يبث **كل عملية مالية على الفاتورة** إلى قروب الواتساب المخصّص، كل فاتورة كرسالة وملف PDF مستقل (قابل للبحث)، + **ملخّص يومي**: أولي الساعة ١ صباحاً ونهائي مُصحَّح الساعة ٥ صباحاً (فقط إذا تغيّر شيء بينهما).

## كيف يشتغل
```
[إنشاء/دفعة/إلغاء/استرجاع/تعديل تسعيرة] في برنامج جسر
        ↓ (trigger في قاعدة البيانات)
   جدول public.wa_outbox  (طابور الإرسال)
        ↓ (هذا البوت كل 20 ثانية: wa_claim_jobs)
   يبني الرسالة + ملف PDF (نفس تصميم فاتورة البرنامج) ويرسلها للقروب
```
- كل حدث = رسالة. الإنشاء + أول دفعة يُدمجان في رسالة واحدة؛ الدفعات اللاحقة كل وحدة رسالتها.
- ملف PDF مرفق مع: الإنشاء، الإلغاء، الاسترجاع، تعديل التسعيرة/الدفعة. الدفعة العادية = نص فقط.
- اسم ملف الـPDF = «العميل-رقم الفاتورة.pdf» ليسهل البحث عنه داخل الواتساب.

> ⚠️ **تنبيه مهم:** يستخدم مكتبة واتساب غير رسمية (whatsapp-web.js). **استخدم رقم/شريحة مكتب مخصّصة — لا تستخدم رقمك الشخصي** (احتمال حظر الرقم قائم). التباعد بين الرسائل (٣–٨ ثواني) والحجم اليومي القليل يقلّلان الخطر.

## متطلبات
- Node.js ≥ 18 على جهاز المكتب (نفس جهاز بوت مقيم).
- مفتاح Supabase **service_role** (من لوحة Supabase → Project Settings → API).

## التشغيل لأول مرة (الإقران)
```bash
cd wa-invoice-bot
copy .env.example .env        REM ثم عبّئ القيم
npm install                   REM يثبّت whatsapp-web.js + puppeteer + supabase (~1-2 دقيقة)
npm start                     REM شغّله في نافذة عادية لتظهر شيفرة QR
```
1. تظهر شيفرة QR في الطرفية → افتح واتساب على هاتف **الرقم المخصّص** → الأجهزة المرتبطة → ربط جهاز → امسح الشيفرة.
2. بعد الربط يطبع البوت قائمة قروباتك. أوقفه (Ctrl+C) ثم:
   ```bash
   npm run list-groups        REM يطبع اسم ومعرّف كل قروب
   ```
3. ضع **WA_GROUP_ID** (الأدق) أو **WA_GROUP_NAME** في `.env`.
4. `npm start` مرة أخرى — يطبع «Ready — posting to group …». جلسة الواتساب محفوظة محلياً (`.wwebjs_auth`) فلا تحتاج QR مرة ثانية.

## التشغيل الدائم (خلفي + إقلاع تلقائي)
بعد نجاح الإقران:
```bash
start.bat                     REM يشغّله مصغّراً في الخلفية، السجل في bot.log
install-autostart.bat         REM (مرة واحدة) يشغّله تلقائياً عند تسجيل الدخول لويندوز
stop.bat                      REM لإيقافه
```

## الإعدادات (.env)
| المفتاح | الوصف |
|---|---|
| `SUPABASE_URL` | رابط المشروع. الافتراضي = الساندبوكس (jisr-dev) للاختبار؛ بدّله للبرودكشن لاحقاً |
| `SUPABASE_SERVICE_ROLE_KEY` | مفتاح service_role (سرّي — لا يُرفع لـGit) |
| `WA_GROUP_ID` / `WA_GROUP_NAME` | القروب الهدف |
| `RIPEN_SEC` | تأخير قبل الإرسال (يسمح بدمج الإنشاء + أول دفعة). افتراضي 60 |
| `SUMMARY_PREVIEW_HOUR` / `SUMMARY_FINAL_HOUR` | ساعتا الملخّص (١ و٥ صباحاً) |
| `SEND_DELAY_MIN/MAX` | التباعد بين الرسائل (مكافحة الحظر) |

## الاختبار (على الساندبوكس)
1. شغّل البوت مربوطاً بقروب تجريبي.
2. في برنامج جسر (وضع الساندبوكس) أنشئ فاتورة، سجّل دفعة، ألغِ/استرجع، عدّل تسعيرة.
3. راقب القروب: تصل رسالة لكل عملية + ملف PDF. السجل في `bot.log`.

## ملاحظات تقنية
- البوت لا يكرّر الإرسال: كل صف في `wa_outbox` يُعلَّم `sent`/`failed`؛ الفشل يُعاد حتى 5 محاولات.
- لو نام الجهاز أو انقطع النت، تنتظر المهام في الطابور وتُرسل عند العودة.
- تصميم الـPDF مصدره الوحيد `src/lib/invoicePrint.js` المشترك مع زر الطباعة في البرنامج — أي تعديل على التصميم ينعكس في الاثنين.
- حدود «اليوم» = ٥ صباحاً بتوقيت الرياض (مطابق للوحة البرنامج تماماً).
