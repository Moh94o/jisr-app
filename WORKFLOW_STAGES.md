# خريطة سير العمل (Workflow Stages) — مسودة أولية للمراجعة

> القرارات المتفق عليها: توجيه **هجين** (دور افتراضي + تعيين يدوي) · **تبويب لكل مرحلة** في السايد بار ·
> تسلسل **صارم** غالباً مع استثناءات قليلة للتوازي · بناء **كل الخدمات** دفعة واحدة.
>
> طريقة المراجعة: عدّل أي صف — اسم المرحلة، الترتيب، الدور المسؤول، أو علامة التوازي.
> **التوجيه التلقائي حسب أدوار الأقسام** (معتمد). الأقسام:

| القسم (الدور) | جديد؟ | المراحل اللي يملكها |
|---|---|---|
| `dept_billing` — الاستقبال والفوترة | موجود («مصدر فواتير») | استلام وفوترة |
| `dept_visa` — قسم التأشيرات | جديد | إصدار التأشيرة · الوكالة · خروج وعودة · خروج نهائي · تحديث الجواز |
| `dept_iqama` — قسم الإقامات | جديد | الفحص الطبي · رخصة العمل · التأمين الطبي · إصدار الإقامة · توصيل الإقامة · طباعة الإقامة |
| `dept_transfer` — قسم النقل وقوى | جديد | طلب نقل الكفالة · موافقة النقل · تغيير المهنة · تعديل الراتب · عقد أجير |
| `dept_docs` — قسم المستندات | جديد | الغرفة التجارية · مستندات · إجراء عام |
| `dept_admin` — الإدارة | موجود («مدير مكتب») | مراجعة وإغلاق |

> «المدير العام» يشوف كل التبويبات. التوجيه هجين: القسم افتراضي، والمشرف يقدر يعيّن مهمة لموظف محدد داخل القسم.

## 3) المخطط التقني (للتنفيذ بعد اعتماد الخريطة)

**جداول جديدة في Supabase:**
- `workflow_stages` — كتالوج المراحل (الجدول رقم 1 فوق): `code, name_ar, name_en, default_role, sort_order`.
- `service_stage_pipelines` — سلسلة كل خدمة: `service_type_code, stage_code, step_order, can_parallel, default_role_override`.
- `transaction_stages` — نسخة مرحلة لكل معاملة: `service_request_id, stage_code, step_order, status(pending/active/done/skipped), assigned_role, assigned_user_id, started_at, completed_at, completed_by, note`.
- `transaction_stage_events` — سجل تاريخي: `transaction_stage_id, event_type, actor_user_id, from_status, to_status, note, created_at`.

**عند إنشاء أي معاملة:** نولّد صفوف `transaction_stages` تلقائياً من الـpipeline، أول مرحلة `active` والباقي `pending`.

**عند الضغط «إنجاز»:** المرحلة → `done` + تسجيل حدث، والمرحلة التالية (أو المتوازية الجاهزة) → `active` فتظهر في تبويب المسؤول التالي.

**السايد بار:** تحت «المعاملات» نعرض تبويب لكل مرحلة فيها معاملات `active`، مع عدّاد (badge) لكل تبويب. التوجيه الهجين: التبويب يعرض المعاملات حسب الدور، والمشرف يقدر يعيّن مهمة لموظف محدد.

**صفحة المهمة:** نفس صفحة تفاصيل المعاملة لكن مركّزة على المرحلة النشطة + يعرض السجل التاريخي (مين سوّى إيش ومتى) فوق.

**الملفات المتأثرة:** `App.jsx` (السايد بار + التوجيه)، `TransactionsPage.jsx` (الكروت تصير مدفوعة بالـconfig بدل التثبيت)، ملف config جديد `src/pages/workflowConfig.js`، migration للجداول، خدمة `src/services/workflowService.js`.
